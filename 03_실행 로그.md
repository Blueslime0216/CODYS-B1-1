# 실행 로그 (대화 로그)

## 1. 시나리오 설명
* **과업**: 비정형 작업 로그 기반 포트폴리오 데이터(JSON) 자동 생성
* **대화 목적**: 개발자(사용자)가 작업한 파편화된 메모와 문서를 투입하여 웹사이트 DB 규격에 맞는 정형화된 JSON 데이터를 얻는 과정 검증.
* **주요 검증 요소 (요구사항 4.7 반영)**:
  - 수치 및 누락 정보 발생 시 AI가 임의로 채우지 않고 역으로 확인 질문을 던지는가 (환각 제어)
  - 중간에 대화가 바뀔 때 이전 문맥을 잃지 않는가 (문맥 유지)
  - 대화 후반부에 추가 정보(GitHub 링크)가 뒤늦게 제공되었을 때, 기존 데이터를 유실하지 않고 정상적으로 반영하는가 (조건 변경 및 추가 정보 처리)
